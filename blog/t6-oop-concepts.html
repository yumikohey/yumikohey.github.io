Object Oriented Programing vs. Functional Programing

According to Smashing Magazine's article, Don't be Scared of Functional Programing , which tells readers that the major difference between Functional Programing and OOP is that Functional Programing is Immutable, and Stateless. First, we have to understand what Immutable and Stateless mean. Immutable means that we cannot change the original variable's data. For instance, an array a = [3,4,5], and we want to push a new element into array a. We'd better create a new array new_a = [3,4,5,6]. On the other hand, Stateless means that the function can only use parameters data to process calculations. No other outside variables is allowed to be used inside the function. This is very different from OOP.

Ruby and JavaScript are Object Oriented Programing, which programmers can write programs with objects / object function. Writing in OOP, programmers don't have to repeat themselves. When programmers decide to generate hundreds of instances, programmers do not have to write repetitive code and instances can inheritate those properties belong to the object immediately. This is the one of the important insights of OOP.

Some of Ruby's methods offer non-destructive and destructive ways, which means we can either be mutable or immutable our variables. On the other hand, we use instance variables inside methods, which means we are using outside variables other than parameters.

That's the differences I notice between OOP and Functional Programing.
